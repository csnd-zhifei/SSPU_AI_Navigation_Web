# 应用流程文档

## 1. 页面加载
1. **加载配置**：
   - 读取 `localStorage` 获取用户设置（显隐状态、引擎开关、Dock 排序）。
   - Fetch 请求获取 `data.json`。
2. **渲染界面**：
   - 根据配置渲染顶部搜索标签（隐藏未勾选的引擎）。
   - 根据配置决定中部标签区是否显示 (`display: none` 或 `block`)。
   - 渲染中部卡片式导航。
   - 渲染底部 Dock 网格。

## 2. 搜索与 AI 交互
- **搜索流程**：输入 -> 判断当前选中标签 -> 跳转搜索结果页。
- **系统响应**：
  1. 标签执行 CSS 上浮动画 + 颜色变化。
  2. 切换当前搜索引擎类型。
- **用户动作**：输入关键词并回车。
- **分支 A (传统搜索)**：
  - 新标签页打开百度/必应/谷歌搜索结果。
- **分支 B (AI 问答)**：
  1. 页面中央（搜索框下方）打开 AI 对话窗口。
  2. 发送请求到后端 `/api/chat`。
  3. **后端逻辑**：
     - 检索本地数据是否匹配关键词。
     - 若匹配：返回指令 `{action: "highlight", ids: [...]}`。
     - 若不匹配：返回指令 `{action: "open_category", target: "..."}` 或文本回答。
  4. **前端处理**：
     - Highlight：对应图标背景变红。
     - Open Category：弹出对应的分类模态框。
     - Ask for Confirm：显示"是否跳转？确认输入y/Y"，等待用户输入 y。

### 2.1 AI 高亮状态管理
- **触发高亮**：当 AI 返回 `action: "highlight"` 时，对应图标背景变红。
- **自动清除高亮**：
  - 用户刷新页面
  - 用户发起新的 AI 对话（新的高亮会替换旧的高亮）
  - 用户切换到其他搜索引擎标签
- **注意**：如果新的 AI 对话没有触发高亮，原来的高亮也会被清除。

### 2.2 AI 对话窗口
- **打开方式**：点击 AI 标签后输入内容，或直接在 AI 模式下输入。
- **关闭方式**：
  - 点击对话框外的遮罩层
  - 按 ESC 键
  - 点击对话框右上角的关闭按钮

## 3. 导航浏览流程
- **用户动作**：点击"学术工具"分类标签。
- **系统响应**：
  1. 屏幕中央弹出**全屏模态框**（带毛玻璃背景）。
  2. 模态框内部显示子分类标签（学术AI、论文网站...）。
  3. 默认显示第一个子分类的图标网格。
- **用户动作**：点击模态框外部区域。
- **系统响应**：关闭模态框。

## 4. 自定义添加流程
- **用户动作**：点击 Dock 栏最后的"+"号。
- **系统响应**：在"+"号旁边弹出小型气泡窗口。
- **用户动作**：填写 URL、名称，选择图标模式，选择"添加到"位置。
- **系统响应**：
  1. 保存数据到 LocalStorage。
  2. 如果是添加到主页，Dock 栏实时新增一个图标。
  3. 如果是添加到分类，更新后台数据结构。

## 5. 右键菜单流程

### 5.1 主页/Dock 区图标的右键菜单
- **用户动作**：右键点击主页或中部导航区的图标。
- **系统响应**：
  1. 阻止默认浏览器右键菜单。
  2. 显示自定义菜单：
     - "添加到主页"
     - "添加到 [分类名]"
     - "删除"
- **用户动作**：点击"删除"。
- **系统响应**：
  1. 弹出确认弹窗："确定要删除 [图标名称] 吗？"
  2. 显示"取消"和"确认删除"按钮。
- **用户动作**：点击"确认删除"。
- **系统响应**：
  1. 从 LocalStorage 中移除该图标数据。
  2. 从界面上移除该图标。
  3. 显示删除成功提示（可选）。

### 5.2 分类弹窗内图标的右键菜单
- **用户动作**：右键点击分类弹窗内的预置图标。
- **系统响应**：
  1. 阻止默认浏览器右键菜单。
  2. 显示自定义菜单：
     - "添加到主页"
     - "添加到 [分类名]"
  3. **注意**：不显示"删除"选项，因为预置图标不可删除。
- **用户动作**：点击菜单项。
- **系统响应**：执行复制操作（添加到目标位置），更新 LocalStorage，提示"已添加"。

## 6. 快捷键监听
- **事件**：全局监听 `keydown`。
- **逻辑**：
  - 判断设置中"快捷键功能"是否开启。
  - 判断当前焦点是否在输入框内（若是则屏蔽快捷键）。
  - 若索引超出范围（1-3），则不执行。
  - 若索引有效，执行跳转。
  - 若按键为 `1`/`2`/`3` -> 跳转 Dock 中对应网站。

## 7. 壁纸管理流程

### 7.1 自定义上传壁纸
- **用户动作**：在设置面板点击"上传壁纸"。
- **系统响应**：
  1. 打开文件选择器（限制图片格式：jpg, png, webp）。
- **用户动作**：选择图片文件。
- **系统响应**：
  1. 发送 POST 请求到 `/api/wallpaper/upload`。
  2. 后端保存图片到 `/static/images/wallpapers/` 目录。
  3. 返回图片 URL。
  4. 前端应用壁纸，保存 URL 到 LocalStorage。

### 7.2 AI 生成壁纸
- **用户动作**：在设置面板点击"AI 生成壁纸"。
- **系统响应**：弹出风格选择窗口（动漫风格、写实风格、抽象风格等）。
- **用户动作**：选择风格，输入描述词（可选）。
- **系统响应**：
  1. 发送 POST 请求到 `/api/wallpaper/generate`。
  2. 后端调用智谱 AI 图像生成 API。
  3. 返回生成图片的 URL。
  4. 前端应用壁纸，保存 URL 到 LocalStorage。

## 8. 错误处理流程
- **网络错误**：显示"网络连接失败，请检查网络后重试"。
- **API 错误**：显示"服务暂时不可用，请稍后重试"。
- **文件上传错误**：显示"文件上传失败，请重试"。
- **AI 响应超时**：显示"AI 响应超时，请重新提问"。
